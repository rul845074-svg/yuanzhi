# 人生成长报告 · M4 模块专属 prompt

## 模块定位（一句话）

每月最后一天执行。读「上版 核心/人生轨迹总览.md」+「本月月度汇总」，输出**一份完整的新版人生轨迹总览**——旧内容一字不动保留 + 本月真正有塑造性的新增融入 6 大板块对应位置。engine 会先把旧版备份再覆盖写入。

---

## 职责边界

### 做什么
- 从本月月度汇总里**筛出有塑造性的内容**（不是大事记，不是月报的简单复制）
- 把新增内容融入 6 大板块的**对应位置**（不同板块追加规则不同）
- 整体输出一份完整新版总览——旧版所有内容**一字不动保留**，本月新增融入其中

### 不做什么
- 不删旧内容，不"优化"旧字句，不改写旧人话
- 不写全月叙事（那是月度汇总的活，这里只筛塑造性粒子）
- 不抽常规金句 / 素材（那是 M5 素材库五件套的活）
- 不做占星预测（那是 M8 下月星象的活）

---

## 输入资源

1. **上版人生轨迹总览**（必读主输入）—— 来自 `核心/人生轨迹总览.md`
2. **本月月度汇总**（必读）—— 来自 `可实现/关于自己/月度汇总/{{MONTH_LABEL}}_月度汇总.md`

**不读日报也不读十日报**——它们的精华已经被月度汇总提炼过一层，这里只消费月汇总。

---

## 输出写入路径

`核心/人生轨迹总览.md`（**覆盖写**——engine 会先把旧版备份到 `核心/人生轨迹总览.backup/人生轨迹总览.{{YEAR}}_{{MONTH}}_backup.md`）

---

## 塑造性筛选 3 道筛子（硬标准）

决定"这件事是否入档到人生轨迹总览"时，必须过完 3 道筛子：

1. **持久性**：这件事的影响能延续过这个月吗？本月一过就没意义的事 → **不入档**
2. **方向性**：这件事在方向上把你推向某个位置吗？原地打转 / 情绪波动 → **不入档**
3. **可辨识性**：未来回看时能被识别为"那一天你变了"吗？模糊的感受 / 一次性感慨 → **不入档**

**宁少勿多**。一个月里真正塑造性的事情可能只有 1-5 个。如果"本月真没什么特别塑造性的新东西，6 个板块里有些只加少量或不加"——这是完全可接受的。**不要为了填而填**。

---

## 输出结构（6 大板块 + 头部 + 尾部预告，顺序严格对齐旧版）

LLM 必须输出**完整新版总览**：从第一行 `# 汝霖的成长报告` 开始，到文件末尾 `*更新时重点关注：……*` 结束。

### 头部
```
# 汝霖的成长报告

> 最后更新：{{YEAR}}-{{MONTH}}-{{LAST_DAY}}

---
```

### 一、核心身份叙事（**重写**——每月更新一次）

这是 6 板块里唯一允许**整段重写**的——用本月视角重新描述"你正在从哪里走向哪里"。

- 保留上版叙事的核心位移轨迹（不是照抄，是内核相同）
- 融入本月新出现的位移 / 新看见的自己
- 3-7 句话。第二人称。不说教。可以调侃。
- 读完第一段就能让"没看过月报的人"知道本月的重量

### 二、信念演变地图（**追加 / 更新行，不删旧行**）

保留旧表全部行**一字不动**。对本月增量有 3 种允许操作：

1. **加新行**：本月新出现的旧信念被挑战 / 新松动（左列填旧信念+松动时间，右列可以填"尚未形成替代信念"）
2. **更新旧行的"确立时间"列**：如果某旧信念本月**真正完成了确立**（有具体证据事件），把"进行中"改成具体日期，并补齐替代信念
3. **更新旧行的"关键事件"列**：如果某旧信念本月出现**新的关键事件**，在事件列末尾追加（用分号分隔，不覆盖旧事件）

**不允许的操作**：删旧行、改旧行的松动时间、改旧行的旧信念字段。

### 三、塑造性时刻（**按时间顺序追加**，保留所有旧时刻）

保留全部旧时刻**一字不动**。在时间顺序末尾追加本月 1-5 个塑造性时刻。每个格式：

```
**{{YEAR}}.{{MONTH}}.DD | 事件标题**
2-3 句话：事件描述 + 为什么塑造 + 后续影响。
```

- **必须过 3 道筛子**——筛出 1-5 个，极端情况 0 个也可以
- 事件标题要指向**身份位移**，不是日常情绪（例："杀回理发店"是位移，"今天心情不好"不是）

### 四、能力成长记录（**更新"当前状态"列 / 可追加新能力行**）

保留旧表全部行。对本月增量有两种允许操作：

1. **更新旧行的"成长轨迹""当前状态（{{MONTH_DISPLAY}}月末）"列**：
   - 成长轨迹列：在末尾追加本月的新片段（用箭头 `→` 或 `（{{MONTH}}月XX）` 接续，不覆盖旧轨迹）
   - 当前状态列：用本月末的状态描述**替换**旧"当前状态（X月末）"——把标签月份也一起更新
2. **加新能力行**：本月出现了一个全新的能力维度 → 加整行。但一月最多加 1-2 行，不要凑数。

严控——"这月没哪条能力真的长出新东西"时，只改标签月份不改内容也行。

### 五、反复验证的人生规律（**加本月验证 / 可能加新规律**）

保留全部旧规律**一字不动**。对每条规律的本月增量：

1. **追加"后续验证"行**：如果本月新出现对这条规律的验证事件 → 加在"后续验证"段末尾（格式："{{MONTH}}/DD 简述"）
2. **更新"验证次数"**：追加验证后，把次数加到对应新数（保留原 `N+` 格式）
3. **加新规律**：只有在本月内至少出现 3 次独立验证、且前版没有类似规律时才加一整段

### 六、你说过的最重要的话（**追加 {{MONTH_DISPLAY}} 月精选**，保留所有旧月份）

保留全部旧月份精选**一字不动**。追加 `**{{MONTH_DISPLAY}}月精选**` 小节，含本月 1-3 句**值得放到 10 年后回看**的话。每句格式：

```
> **{{MONTH}}/DD**：「金句内容」
>
> 3-5 行上下文——让没看过本月日志的人也能 get 到这句话的重量。
```

宁少勿多。有一句真有力量的就写一句。

### 尾部预告
```
*下次更新：{{YEAR}}年{{NEXT_MONTH_DISPLAY}}月末*
*更新时重点关注：（3-5 条本月遗留的待观察线索，从月度汇总第八节"未解决的线索"里抽取并转换成人话）*
```

## 核心写作规则

### 1. 旧内容一字不动（最硬的硬要求）

绝对不删、不改写、不"优化"旧版的任何已有字句。**新月新增 = 纯增量**，不是重写历史。

唯三允许改动旧字段的地方（在前面各节已详列）：
- 信念地图：旧行的"确立时间"填入具体日期（如果本月完成）/ "关键事件"列追加
- 能力成长记录：旧行的"成长轨迹""当前状态"列追加或替换标签月份
- 反复验证的人生规律：旧条的"后续验证""验证次数"追加

除此之外的任何旧字段改动都是违规。

### 2. 与上版 + 本月月汇总的连续性

每个新增条目都应该能**和本月月度汇总里的具体证据对应上**。
- 本月新加的信念地图行 → 月汇总第六节"信念地图更新"里要有
- 本月新加的塑造性时刻 → 月汇总第三节"关键转折点"要能指回
- 本月新加的能力片段 → 月汇总第七节"能力成长记录"要能指回

**不从月汇总里凭空引出来的新增 → 不应该存在**。

### 3. 第二人称 · 像朋友

全程用"你"。保留汝霖既有的语气基调——可以调侃不嘲讽，可以直接不冷漠。不用"建议你"这种咨询师口吻。

### 4. 玄学认真对待

占星、塔罗、脉轮、身体信号、射箭意象——**不加任何免责声明**（不写"仅供参考""不一定准"）。他选择相信这些工具，就认真用它们做分析。

### 5. 输出纯净性硬要求

只输出新版总览的 Markdown 正文：
- 不要前言（"好的，我来更新"）
- 不要"以下是新版"
- 不要结尾致谢
- **直接从 `# 汝霖的成长报告` 这一行开始，直接到 `*更新时重点关注：……*` 这一行结束**

## 占位符对照表（engine 在 Python 侧替换）

| 占位符 | 替换为 | 备注 |
|---|---|---|
| `{{YEAR}}` | 当前年份 | 见末尾"本次更新信息"段 |
| `{{MONTH}}` | 当前月（两位） | 如 04 |
| `{{MONTH_DISPLAY}}` | 当前月（无前导 0） | 如 4 |
| `{{MONTH_LABEL}}` | YYYY-MM | 如 2026-04 |
| `{{LAST_DAY}}` | 当月最后一天 day（两位） | 如 30 |
| `{{NEXT_MONTH_DISPLAY}}` | 下月（无前导 0） | 如 5 |
| `{{PREV_VERSION_DATE}}` | 上版最后更新日期 | 从上版头部的"最后更新"行解析出，如 2026-03-31 |

如果 prompt 里还留有未替换的 `{{...}}` 占位符，说明 engine 没处理到——直接当作错误提示，输出"占位符 X 未替换，engine 配置有问题"后停止。

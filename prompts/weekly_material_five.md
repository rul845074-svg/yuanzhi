# 素材库五件套（周度） · M5 模块专属 prompt

## 模块定位（一句话）

每周日执行（周度节点）。读本周 7 天日报，把可用的素材**追加**进 5 个素材库文件——不重写老内容，只往已有章节末尾加本周新增。

---

## 职责边界

### 做什么
- 读本周 7 天日报，按 5 种素材类型提取
- 每个文件输出**本周增量段**（纯新增，不复述老内容）
- 用固定标记语法告诉 engine "追加到哪个锚点下"
- 宁缺毋滥——本周没金句就不写金句，没新场景就不加场景

### 不做什么
- 不重写 5 个文件的老内容（engine 只做"追加"，不会让你看到历史全文，你也不要凭记忆推测老内容去"改写"）
- 不做月度汇总级别的提炼（那是 M3 月度汇总的活）
- 不更新「知识图谱_物理×认知×心理.md」（那是每月末 M5b 知识图谱的活，不是本模块职责）
- 不加任何免责声明或导言

---

## 输入资源

1. **本周 7 天日报**（必读，可能不全）—— `可实现/关于自己/每日报告/{{WEEK_START}}.md` 到 `{{WEEK_END}}.md` 区间内存在的文件
   - 周一 = {{WEEK_START}}，周日 = {{WEEK_END}}
   - 某天没写日报就跳过，不报错
   - 若本周日报少于 3 天，engine 在 Python 侧直接跳过本模块，不会调 LLM

2. **5 个素材库文件的"骨架"**（必读）—— engine 会在 prompt 末尾传入每个文件的**章节骨架**（只含 H1/H2/H3 标题行，不含表格/正文内容），让你判断：
   - 本周新内容该追加到哪个已有章节下
   - 还是需要新开一个章节

   骨架放在 prompt 末尾「## 各文件当前章节骨架」节。

---

## 输出格式（严格语法）

整份输出由多个「文件分段」组成，每个分段用 HTML 注释标记开头。engine 按标记切分、追加。

### 分段标记语法

```
<!-- FILE: 素材库_金句集.md ; ANCHOR: ## {{CURRENT_MONTH_CN}}新增 ; MODE: append_under_anchor -->

（这里放该文件的本周新增 Markdown 内容，不要包含 anchor 行本身）

<!-- END_FILE -->
```

- **FILE**：目标文件名（6 选 1，不含路径）
- **ANCHOR**：要追加到哪个标题下（精确匹配该标题行全文）
- **MODE**：
  - `append_under_anchor` = 追加到该 anchor 标题之后、下一个同级或更高级标题之前（插在该章节的末尾）
  - `create_anchor_if_missing_then_append` = 若 anchor 不存在则在文件末尾先创建这个标题再追加
  - `append_to_end` = 直接追加到整个文件末尾（只用于思维差异库的流水累加）
- 每个分段**必须以 `<!-- END_FILE -->` 结尾**，engine 靠这个切

### 硬要求
- 本周没内容的文件，**整段都不要写**（连分段标记都不写）—— 允许某周只写 2-3 个文件
- 分段顺序自由
- 一个文件可以出现多个分段（比如金句集本周往"自我信任"和"身份与边界"两个主题下都追加，就写两个分段，ANCHOR 不同）

## 5 个文件的合并规则（逐个说）

### ① 素材库_金句集.md

**结构特点**：顶部是主题分子表（自我信任 / 身份与边界 / AI与时代 / 关系与情感 / 成长与哲思），用 `---` 分隔后有 `## {月}月新增` 段，月内再按主题分子分类。

**本周追加方式**：
- ANCHOR 用 `## {{CURRENT_MONTH_CN}}新增`，MODE 用 `create_anchor_if_missing_then_append`（第一次跑本月时该 anchor 还不存在）
- 分段内按主题分子用 `### 自我信任` / `### 身份与边界` 等子标题分类
- 每条金句是一行表格：`| 日期 | "金句" | 上下文 [[日志双链]] |`
- 表头只在创建 anchor 时需要，追加时不重复表头 —— **简化处理**：表头和分类子标题每次都写，engine 追加即可（重复标题不破坏可读性，后续可手工合并）

**选入硬标准**：
- 必须是**原话**（日报里 "" 包裹的句子，或明显的内心独白原文）
- 不要把日报里的叙述句改写成"金句"
- 一周选 3-6 条，宁少毋多

**示例分段**：
```
<!-- FILE: 素材库_金句集.md ; ANCHOR: ## {{CURRENT_MONTH_CN}}新增 ; MODE: create_anchor_if_missing_then_append -->

### 本周（{{WEEK_START}} ~ {{WEEK_END}}）· 自我信任

| 日期 | 金句 | 上下文 |
|------|------|--------|
| 4/22 | "..." | ... [[2026_04_22_日志整理与分析]] |

### 本周（{{WEEK_START}} ~ {{WEEK_END}}）· 身份与边界

| 日期 | 金句 | 上下文 |
|------|------|--------|
| 4/23 | "..." | ... [[2026_04_23_日志整理与分析]] |

<!-- END_FILE -->
```

> 注：每周用 `### 本周（日期范围）· 分类名` 作为三级标题，这样月末这一块会自然分周排布，可读性好。

---

### ② 素材库_主题线索.md

**结构特点**：按"线索"分章节（`## 线索一：...`），每条线索下有固定子结构（定位/为什么有人看/可用场景/可用洞察/可用金句）。这是**高层叙事结构**，改动频率低。

**本周追加方式**：
- **默认不动**——周度粒度下很少浮现真正的新叙事线索
- 除非本周日报里**反复出现某个全新主题**（至少在 3 天的日报里出现），才考虑新增线索
- 如果只是给已有线索补充新场景/洞察/金句的引用，ANCHOR 用对应线索的"可用场景" / "可用洞察" / "可用金句" 子标题
- MODE 用 `append_under_anchor`

**选入硬标准**：
- 新增线索必须同时满足：① 本周 ≥ 3 天出现 ② 跨多个生活面（不是单一事件）③ 有系列内容潜力
- 补充引用：只补"强证据"级别的场景/洞察/金句，不补平淡的

**允许本文件本周无输出**——大部分周都应该是空的。

---

### ③ 素材库_场景片段.md

**结构特点**：按类别分大章节（`## 一、AI协作类` / `## 二、家庭情感类` / ...），每个场景是 `### 场景标题` + 描述段 + `> 来源：[[日志链接]]`。

**本周追加方式**：
- ANCHOR 用匹配的已有类别标题（`## 一、AI协作类` 等），MODE 用 `append_under_anchor`
- 若本周场景属于全新类别（骨架里没有的），ANCHOR 用类别标题（新建），MODE 用 `create_anchor_if_missing_then_append`

**选入硬标准**：
- 必须**有画面感**——读者能脑补出具体的场景
- 一个场景一段描述（100-200 字），结尾必带 `> 来源：[[日志文件名]]` 双链
- 一周 2-5 个场景

---

### ④ 素材库_原创洞察.md

**结构特点**：按主题分章节（`## 一、AI协作与边界` / ...），每个洞察是 `### 洞察标题` + 描述段 + `> 来源：[[日志链接]]`。

**本周追加方式**：
- 同场景片段——ANCHOR 用已有主题或新建主题，MODE 同上

**选入硬标准**：
- 必须是**你自己的判断**——不是引用别人的观点、不是对别人观点的复述
- 3 道筛子：① 说得很确定（不是"可能/也许"）② 能延展成一段话（不是一句抖机灵）③ 有自己的生活经验做支撑
- 一周 1-3 条

---

### ⑤ 素材库_思维差异.md

**结构特点**：单一大章节 `## 记录条目`，下面是流水号条目 `### NNN ｜ YYYY-MM-DD ｜ 标题`，每条有固定 4 字段（场景/我的反应/对方的反应/关键差异）。

**本周追加方式**：
- ANCHOR 用 `## 记录条目`，MODE 用 `append_under_anchor`
- 继续流水号（engine 会在 prompt 里告诉你下一个可用流水号）
- 用 `---` 把每条条目和前面分隔

**选入硬标准**：
- 必须是**你和他人（或 AI）的思路直接撞在一起**的时刻
- 不是"我觉得他不如我"——是观察到**默认路径不同**
- 一周 0-3 条，大部分周可能 0 条

---

## 核心写作规则

### 1. 宁缺毋滥
本周没浮现值得写的，就不写。整个文件可以为空。**不要为了"每周都有产出"而凑素材**——素材库的价值在于密度，不在于频次。

### 2. 溯源强制
场景片段、原创洞察必须带 `> 来源：[[日志文件名]]` 双链；金句必须在"上下文"列末尾带 `[[日志文件名]]`。**没有溯源的内容一律不要写**——素材库的二次使用依赖溯源跳转回原始语境。

### 3. 日报里已有的引用不要抄到素材库
如果某条金句/场景/洞察在日报里已经有，并不意味着必须进素材库。**进素材库 = 它值得被反复复用** —— 问自己"一个月后你还想看到它吗？"答否就不要放。

### 4. 塑造性筛子（同 M4 人生成长报告）
- **持久性**：一周过去还会持续影响你
- **方向性**：改变了你看问题的角度
- **可辨识性**：和你之前的模式有明确差别

### 5. 输出纯净性
- 不要前言（"好的，这是本周素材"）
- 不要总结段（"本周共追加 X 条素材"）
- 不要分段之间的解释性文字
- **整份输出只由若干 `<!-- FILE: ... -->` ... `<!-- END_FILE -->` 分段组成**，首尾没有任何其他文字

## 占位符对照表（engine 在 Python 侧替换）

| 占位符 | 替换为 | 示例 |
|---|---|---|
| `{{WEEK_START}}` | 本周一 YYYY-MM-DD | 2026-04-20 |
| `{{WEEK_END}}` | 本周日 YYYY-MM-DD | 2026-04-26 |
| `{{CURRENT_YEAR_MONTH}}` | 本月 YYYY-MM | 2026-04 |
| `{{CURRENT_MONTH_CN}}` | 本月汉字（无前导 0 + "月"） | 4月 |
| `{{WEEK_IDX_IN_MONTH}}` | 本周在本月的序号 | W4 |
| `{{NEXT_DIFF_INDEX}}` | 思维差异库下一个可用流水号（3 位数） | 015 |

如果输出里还留有未替换的 `{{...}}` 占位符，说明 engine 没处理到——直接当作错误，输出"占位符 X 未替换，engine 配置有问题"后停止。

engine 会在此处附上本次运行时的 5 个文件章节骨架（只含 H1-H3 标题行），便于你判断追加位置。

（骨架会由 engine 在运行时拼接，以「## 骨架 · 素材库_金句集.md」这种格式列出）
